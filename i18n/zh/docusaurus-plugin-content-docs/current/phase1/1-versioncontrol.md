# 主题 1：版本控制

> ⏱️ **预计时间：** 3-5 天

在 L2C，我们使用 Git 和 GitHub 进行版本控制。你需要花几天时间来熟悉它们。

---

## 📚 学习路径

1. **学习：** [Git 入门](https://learn.microsoft.com/training/modules/intro-to-git/)

2. **创建 [GitHub 账号](https://docs.github.com/get-started/start-your-journey/creating-an-account-on-github)**

3. **给 [Learn to Cloud 仓库](https://github.com/learntocloud/learn-to-cloud)点亮 Star**

4. **设置 AI 学习助手**
   
   在 L2C，我们非常推崇生成式 AI（GenAI）辅助学习的能力。我们经常利用它来深入探索和澄清概念。现在是时候为你自己喜欢的工具注册，并开始将其用于日常学习的好时机。
   
   先试着通过测试你对版本控制的理解来体验一下。可以使用以下提示词（Prompt）作为模板：

    ```txt
    我正在学习云工程，最近了解了版本控制。我将向你解释它的概念，如果我的解释不清楚，请向我提问。我想确保自己真正理解了这个概念，所以请不要直接纠正我，只需不断提问，直到我给出的解释完全正确。以下是我的解释：版本控制是……
    ```

5. **[安装 VS Code](https://code.visualstudio.com/)** —— 我们所有的编程工作都在 VS Code 中完成

6. **仅限 Windows 用户：** [设置 WSL](https://learn.microsoft.com/windows/wsl/install)
   
   我们所有的实验和工具都需要基于 Linux 的环境，WSL 将为你提供这一环境。然后请参考：[在 Windows Subsystem for Linux 中开始使用 Visual Studio Code](https://learn.microsoft.com/en-us/windows/wsl/tutorials/wsl-vscode)

7. **学习命令行基础：** [命令行初学者指南 —— 像专业人士一样使用终端](https://www.freecodecamp.org/news/command-line-for-beginners/#heading-most-common-and-useful-commands-to-use)

8. **创建你的工作空间**
   
   使用终端，在你的计算机上创建一个文件夹，用于存放所有的实验和项目。将其命名为 `l2c`。请使用 `mkdir` 和 `cd` 命令来完成此操作。

9. **克隆实验仓库**
   
   在你的 `l2c` 文件夹中运行：`git clone https://github.com/learntocloud/linux-ctfs`

10. **学习 Markdown：** [使用 Markdown 进行交流](https://learn.microsoft.com/en-us/training/modules/communicate-using-markdown/)

11. **创建你的 GitHub 个人主页 README：** [添加个人主页 README](https://docs.github.com/en/account-and-profile/setting-up-and-managing-your-github-profile/customizing-your-profile/managing-your-profile-readme#adding-a-profile-readme)

12. **练习 Git 工作流**
    
    将你的 README 仓库克隆到本地计算机。使用 VS Code 进行编辑，并通过终端将更改推送回 GitHub。这里有一份关于[在 VS Code 中使用 Markdown 的绝佳指南](https://www.freecodecamp.org/news/how-to-use-markdown-in-vscode/)。不要觉得非要把 README 做得非常花哨 —— 简单介绍一下你自己，并添加你的社交媒体链接即可。
